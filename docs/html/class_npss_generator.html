Подробное описание

Генератор NPSS (Narrowband Primary Synchronization Signal) последовательности

Генерирует NPSS последовательность для первичной грубой синхронизации в NB-IoT. Последовательность строится на основе последовательности Задова-Чу:

\[ d_l(n) = S(l) \cdot e^{-j \frac{\pi u n(n+1)}{11}}, \quad n = 0, 1, \ldots, 10 \]

где:

  • \( u = 5 \) — корневой индекс последовательности Задова-Чу
  • \( S(l) \) — множитель, зависящий от индекса OFDM-символа \( l \) (Table 10.2.7.1.1-1)
  • \( n \) — индекс элемента внутри одного OFDM-символа

Полная NPSS последовательность длиной 121 элемент формируется из 11 OFDM-символов по 11 элементов в каждом.

См. также
3GPP TS 36.211 V17.0.0, Section 10.2.7.1.1 "Sequence generation"
https://portal.3gpp.org/desktopmodules/Specifications/SpecificationDetails.aspx?specificationId=2425

◆ getNpssSequence()

const std::array< std::complex< float >, 121 > & NpssGenerator::getNpssSequence ( ) const

Возвращает сгенерированную NPSS-последовательность.

Последовательность вычисляется при первом вызове и кэшируется на всё время жизни программы (static const + лямбда-инициализация).

Формула (3GPP TS 36.211, 10.2.7.1):

\[ d_l(n) = S(l) \cdot e^{-j \frac{\pi u n(n+1)}{11}}, \quad n = 0, 1, \ldots, 10 \]

где:

  • \( u = 5 \) — корневой индекс последовательности Задова-Чу
  • \( S(l) \) — множитель, зависящий от индекса OFDM-символа \( l \) (Table 10.2.7.1.1-1)
  • \( n \) — индекс элемента внутри одного OFDM-символа

Константная ссылка на std::array из 121 комплексных отсчётов.
